# Journey (Hero → About) — page override

> Overrides `design-system/MASTER.md`. Establishes the **dark space-journey** pattern. Component: `src/components/Journey.tsx` (the bright Hero and the About space scene are **fused into one pinned stage**). `src/components/About.tsx` is the older standalone version — now **unused/superseded** by Journey.

## Concept
The Hero and About share ONE pinned, scroll-scrubbed stage layered as: **Hero (bottom) + space/planet/copy overlay (top)**. You start on the hero; as you scroll (page held/pinned, NOT scrolling to a new section) the **origin planet pops up from the centre of the hero**, grows, and **deep space washes over the hero in place** (the hero zooms past + fades) — the About scene covers/fills the hero. The planet then zooms and "lands" and the copy reveals. Fully reversible (scrub) — scroll up and the planet shrinks back into the hero and the cat returns. This is the tonal pivot: dark space from here on.

## Mechanic (GSAP ScrollTrigger, CDN globals)
- One pinned `100svh` stage holding `.jr-hero` (the `<Hero/>` layer) + `.jr-overlay` (space/planet/copy, inline `opacity:0` until JS so it never flashes over the hero). **The overlay starts `pointer-events:none`** (and a ScrollTrigger `onUpdate` flips it to `auto` only past progress 0.42) — otherwise the invisible overlay sits on top of the hero and eats all clicks/hovers/text-selection. Critical for any future "scene-over-scene" layering.
- `ScrollTrigger { pin:true, scrub:1, pinSpacing:true, anticipatePin:1, invalidateOnRefresh:true, end:()=>'+=' + innerHeight*(mobile?1.6:2.2) }`.
- Timeline (ease `none`) — **background-first order**: space fades in over the hero (0.0) + `.jr-hero` `autoAlpha 0 + scale 1.12` zooms past & hides (0.04, so its buttons can't be clicked under About) → THEN planet pops `scale 0→0.5` from centre on the now-dark space (0.3) · grows to 1 (0.46) · **zoom/land** scale→`mobile?1.25:1.7` + drift desktop x:+0.2vw / mobile y:−0.17vh (0.7) · scrim (0.70) · **copy masked slide-up reveal** (0.72): every `.about-line` sits in an `overflow-hidden` wrapper and **wipes up from behind it** (`yPercent 120→0` + fade, staggered) — a premium line-by-line reveal; the heading word (`.about-gradient`) has a flowing animated gradient (gated to `.about-active`, off for reduced-motion). The copy is intentionally **large/editorial** (heading `text-5xl sm:text-7xl`, body `sm:text-xl`, `max-w-xl`); short viewports (`max-height` media queries) clamp it down so the pinned stage never clips. The heading's neon glow was dropped for a cleaner premium look (and so the mask doesn't clip it).
- **Function-based** scale/x/y (+`invalidateOnRefresh`) so resize/rotate recalc. `ScrollTrigger.config({ ignoreMobileResize:true })`. `gsap.context().revert()` = StrictMode-safe.
- **Reduced motion:** no pin — renders `<Hero/>` then a static landed About `<section>` below (sequential, legible).

## Layout / legibility
- **Desktop:** planet drifts right, copy left (vertically centered), left-fade scrim.
- **Mobile:** planet drifts up (smaller), copy stacked in the lower third (`items-end`), bottom-fade scrim.
- Two `.about-scrim` layers (`hidden sm:block` / `sm:hidden`), both GSAP-faded.
- **Short viewports** (`@media max-height:740/560`): `.about-copy` h2/p scale down (clamp+vh) so the pinned, overflow-hidden stage never clips copy.

## 3D planet (`Planet3D.tsx`)
The planet is a **real WebGL sphere** (three.js), not the flat image — for motion users. `AboutLayers` takes a `use3D` prop: Journey passes `use3D` (3D), the reduced-motion static path renders the `<img>`. Scene: dark lit sphere + cyan/magenta **fresnel-rim ShaderMaterial** + additive geodesic **wireframe network** + glowing **Points** nodes + tilted **orbiting ring** + soft glow sprite. Spins continuously via a `requestAnimationFrame` loop. Lives inside the GSAP-scaled `.about-planet` container, so the scroll pop/grow/land still drives its size/position (canvas is CSS `width/height:100%`).
- **three.js is dynamic-imported** (`await import('three')`) → separate ~190 KB-gzip lazy chunk; main bundle stays ~68 KB gzip.
- **Lifecycle/perf (post-review):** render loop paused off-screen (IntersectionObserver) + tab-hidden; renderer sized from `clientWidth*MAX_SCALE` (crisp when CSS-scaled to land) with `setSize(w,h,false)` to keep CSS 100%; `forceContextLoss()` on cleanup (no leaked GL contexts under StrictMode/remounts); `webglcontextlost` → swap to `<img>`; `powerPreference:'default'`; StrictMode-safe via a `cancelled` flag on the async import. Reduced-motion / WebGL-unsupported / import-failure all fall back to `origin-planet-cutout-opt.webp`.
- **Soft-skill constellation:** the 7 soft skills (`SKILLS` export) are bright **nodes on the spinning sphere** (front-hemisphere lat/lon). **Hovering the actual planet SPHERE freezes the spin** — hit-tested by **raycasting** the body mesh (`hitsPlanet(clientX,clientY)`: canvas-rect → NDC → `Raycaster.intersectObject(bodyMesh)`), NOT the square `.about-planet` container, so hovering the empty corners/edges of the square does nothing (this was the fix — the old div-`pointerenter` froze on the whole box). When engaged the spin eases to the nearest `2π` (nodes face camera, "skills pose") + reveals HTML **label chips** at each node's `camera.project()` point with **SVG connector lines** (positioned imperatively in the rAF — no React churn). Because chips are fanned *outside* the sphere, moving onto one would drop the raycast hover, so **`chipHoverCount`** (per-chip `pointerenter`/`pointerleave`) keeps it engaged while you read a chip's description (CSS `group-hover`); `engaged = hover || pinned || chipHoverCount>0`. Mouse uses the raycast hover (cursor → `pointer` only over the sphere); **touch uses tap-to-toggle `pinned`** (`pointerup` + `hitsPlanet`, since there's no hover).
- **Reach/timing:** a **480 ms grace window** (`engaged = active && (rawEngaged || now-lastEngaged < GRACE_MS)`) bridges the dead zone between the sphere and a fanned-out chip, and `labelAlpha` fades **in fast / out slow** (≈`dt*6` vs `dt*1.7`) so chips stay grabbable (`pointerEvents:auto` at `labelAlpha>0.2`) ~1.3 s after you leave — otherwise the labels vanished before you could hover a skill.
- **`active`/un-freeze:** `engaged` is gated on a throttled (~7×/s) `landed()` check; when not landed it **clears `hover`/`pinned`/`chipHoverCount`**, so scrolling away always resumes the spin and never re-lands already-frozen (fixes the stale-hover/overlay-`pointerEvents:none`/touch-pin-sticks edge cases an adversarial review caught).
- **`webglcontextlost`** now calls `cleanup()` directly (deps `[]` means React won't run effect cleanup on the `setFailed` re-render) — no leaked RAF loop / observers / GL context.
- **Chip cards (premium):** each chip is a refined **glass pill with a muted lucide icon** (`SKILL_ICONS` map); hover expands a **glass detail card** whose icon/name/description **fade + slide in with a staggered delay** (`delay-75/100/150`). Low chips flip the card **upward** (`.skill-up .skill-detail` toggled in `positionLabels` when `cy>58`); width `w-48` so it never clips L/R.
- **Counter-scale (critical):** the chips live inside `.about-planet`, which GSAP scales to `LAND_SCALE` (1.7 desktop / 1.25 mobile) — that blew up the chip text/cards by the same factor. The labels overlay (`labelsRef`) is **counter-scaled `1/LAND_SCALE`** (set in the effect) so chips render **1:1/crisp**, and `positionLabels` remaps every coord by `m(p)=50+(p-50)*LAND_SCALE` so they still land on the right screen point (math: the two scales cancel). Pill text is now ~12px on screen (was ~20px).
- **z-index:** `.about-planet` is `z-20` (above the `.about-copy` container's `z-10`) so the left chips/cards (Critical Thinking, Teamwork) **render over the copy instead of behind it** when revealed. Gated to `landed()` (`>360px` rect). `sr-only` list for AT; reduced-motion path shows a static chip list in `.about-copy`.

## Assets / perf
- `origin-planet-cutout-opt.webp` (221 KB, from the 1.66 MB PNG via ffmpeg-static, `scale=1100 -c:v libwebp -quality 90`) — now the **fallback** image (reduced-motion / no-WebGL). `decoding="async"`, intrinsic 1100×1100.
- Reduced-motion: early-return, no pin/scrub; CSS defaults render the static legible "landed" view (`.about-planet` has a responsive CSS transform; space/scrim/copy default to opacity 1). Idle loops (`twinkle`, `planetBob`) are `animation:none`.
- Idle loops are **gated** to on-screen via `.about-active` (ScrollTrigger `onToggle`) — `animation-play-state: paused` otherwise.

## Cross-cutting fixes made here (apply project-wide)
- Removed `scroll-behavior:smooth` (`scroll-smooth`) from `<html>` — it fights ScrollTrigger pin/scrub. Use JS/GSAP `scrollTo` for nav anchors.
- `overflow-x-hidden` → **`overflow-x-clip`** on `<body>`/`<main>` (clip doesn't force `overflow-y:auto`, avoiding the pin-ancestor scroll-container gotcha).
- Section has `id="about"` + `aria-labelledby="about-heading"`.
